# kodex

AI knowledge graph that learns across sessions. Builds a persistent, queryable graph from your codebase and accumulates knowledge as you work — patterns, decisions, conventions, domain concepts — so the next session starts where the last one left off.

Single Rust binary. HDF5 storage. Tree-sitter AST extraction. MCP server for AI editors.

## Install

```bash
cargo install --path .
# or
cargo build --release
cp target/release/kodex ~/.local/bin/
```

## Quick Start

```bash
kodex run ./my-project              # build knowledge graph
kodex query "how does auth work"    # search the graph
kodex path "Client" "Database"      # find connection path
kodex explain "AuthService"         # show node details
```

## How It Works

```
                    kodex.h5 (single file, source of truth)
                   ┌─────────────────────────────────────┐
                   │  /nodes   - code structure (AST)    │
                   │  /edges   - relationships           │
                   │  /knowledge - AI-accumulated wisdom  │
                   └──────┬──────────────┬───────────────┘
                          │              │
                    kodex query     kodex vault
                    kodex serve     (on-demand)
                    (fast read)         │
                          │              ▼
                          │         vault/*.md
                          │         (Obsidian / human reading)
                          ▼
                    AI Editor (Claude, Cursor)
                    reads/writes knowledge
```

**kodex.h5** is the single source of truth. It contains:
- Code graph: classes, functions, imports, call relationships (from AST)
- Communities: Louvain-detected clusters
- Knowledge: patterns, decisions, conventions accumulated by AI across sessions

**vault .md files** are generated on-demand for human browsing in Obsidian.

## Obsidian Vault

Vault is generated during `kodex run` and lives at `kodex-out/vault/`. Open it in Obsidian for visual exploration with wikilinks, community maps, and Dataview queries.

To regenerate vault from existing data:

```bash
kodex run ./my-project    # full pipeline (includes vault generation)
```

## AI Knowledge Accumulation

AI agents accumulate knowledge directly in `kodex.h5` via MCP tools. No `.md` files needed for the knowledge loop.

### How It Works

```
Session 1: Claude works on code
  → discovers "Repository pattern for DB access"
  → MCP learn() → kodex.h5 /knowledge/ (confidence: 60%)

Session 2: Claude starts, MCP knowledge_context()
  → reads knowledge from kodex.h5 directly
  → "Repository Pattern (60%, 1 observation)"
  → discovers same pattern in another module
  → MCP learn() → reinforces: observations 2, confidence 68%

Session 10:
  → confidence 89%, 8 observations → established knowledge
```

### Setup

Add to your project's `CLAUDE.md`:

```markdown
## Kodex

### Session start
Run `kodex serve` (MCP) and call `knowledge_context` to load accumulated knowledge.
Or read `kodex-out/GRAPH_REPORT.md` for project structure overview.

### During work
Use MCP `learn` tool to save knowledge worth remembering for next sessions.
Use MCP `recall` tool to search existing knowledge.
```

### Knowledge Types

| Type | When to Save | Example |
|------|-------------|---------|
| `architecture` | System structure | "3-layer: API → Service → Repository" |
| `pattern` | Design pattern | "Observer for event handling" |
| `decision` | Design choice + reason | "JWT for stateless microservices" |
| `convention` | Code convention | "All errors wrapped in AppError" |
| `coupling` | Module dependency | "auth changes require session changes" |
| `domain` | Business concept | "Trade states: pending → filled → cancelled" |
| `preference` | User style | "Prefers functional over OOP" |
| `bug_pattern` | Recurring bug | "Off-by-one in pagination" |
| `tech_debt` | Improvement needed | "Legacy auth needs rewrite" |
| `ops` | Deploy/infra | "Staging uses different DB creds" |
| `performance` | Bottleneck | "N+1 query in user listing" |
| `lesson` | Learned from mistake | "Don't mock DB in integration tests" |

Any type not listed — invent one. AI decides what's worth saving.

### Confidence Growth

Repeated observations increase confidence asymptotically:

```
Obs 1: 0.60 → Obs 2: 0.68 → Obs 3: 0.74 → Obs 5: 0.83 → Obs 10: 0.93
```

## Storage

`kodex.h5` is a single [HDF5](https://crates.io/crates/rust-hdf5) file (pure Rust, no C dependency):

```
kodex.h5
├── /nodes/          ← code graph nodes (vlen strings)
│   ├── id, label, file_type, source_file, confidence
│   └── community    (u32)
├── /edges/          ← relationships
│   ├── source, target, relation, confidence
│   └── weight       (f64)
└── /knowledge/      ← AI-accumulated knowledge
    ├── titles, types, descriptions, related, tags
    ├── confidence   (f64)
    └── observations (u32)
```

## MCP Server

```bash
kodex serve
```

| Tool | Description |
|------|-------------|
| `query_graph` | BFS/DFS traversal from keyword-matching nodes |
| `get_node` | Fetch node details by label |
| `god_nodes` | List most-connected entities |
| `graph_stats` | Node/edge/community counts |
| `learn` | Store/reinforce knowledge in kodex.h5 |
| `recall` | Query knowledge by keyword or type |
| `knowledge_context` | Compact summary for session bootstrap |
| `save_insight` | Link multiple nodes with a named pattern |
| `save_note` | Free-text memo with related nodes |
| `add_edge` | Add relationship between nodes |

## Architecture

```
kodex run
  ├─ detect      Find code/doc files, skip sensitive, apply .kodexignore
  ├─ extract     Parse AST with tree-sitter → nodes + edges
  ├─ build       Assemble into petgraph directed graph
  ├─ cluster     Louvain community detection
  ├─ analyze     God nodes, surprising connections, questions
  └─ export      kodex.h5 + graph.html + vault/*.md + GRAPH_REPORT.md

kodex serve (MCP)
  ├─ query_graph, get_node, god_nodes, graph_stats
  ├─ learn, recall, knowledge_context
  └─ save_insight, save_note, add_edge

kodex.h5 = code graph + knowledge (single file, source of truth)
vault/   = on-demand Obsidian export (human-readable view)
```
